# Code of Conduct

Quorvex AI follows a simple standard for project participation: be respectful, constructive, and focused on improving the software and documentation.

## Expected Behavior

- Treat contributors and users with respect.
- Assume good intent, but be direct when discussing technical issues.
- Keep discussions focused on the project, its users, and maintainable engineering outcomes.
- Give actionable feedback with enough context for someone else to respond or fix the issue.
- Respect maintainers' decisions about scope, security, roadmap, and release timing.

## Unacceptable Behavior

- Harassment, threats, personal attacks, or discriminatory language.
- Publishing private information without permission.
- Repeated off-topic comments after a maintainer asks to refocus.
- Security disclosure in public issues before maintainers have had a reasonable chance to respond.

## Reporting

Report conduct concerns privately to the project maintainer through the contact method listed on the maintainer's GitHub profile, or open a GitHub issue if the concern is not sensitive.

Maintainers may remove comments, close issues, block users, or take other moderation actions when needed to keep the project usable for contributors and users.
